# Git 실습
### VS CODE에서 Git 명령어를 실습하고 있습니다.
### Git은 파일 자체가 아니라 변경사항을 의미있는 버전 단위로 관리한다.

#### git status before git add ...: Changes not staged for commit:
#### git status before git add ...: Changes to be committed:

#### git log --oneline: 커밋 기록을 확인

### Stage는 이번 버전에 어떤 변경 사항을 넣을지 선택하는 공간
### Commit은 Stage에 올린 변경사항들을 하나의 버전으로 저장하는 단계
### Push는 로컬 컴퓨터에서 만들어진 commit을 GitHub으로 전송하는 단계인데 중요한 점은 
### Git은 파일 자체가 아니라 Commit을 Push한다는 것입니다. 따라서 수정한 파일이 있어도 
### Commit하지 않았다면 Push할 대상이 없습니다.
